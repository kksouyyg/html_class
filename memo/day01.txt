
웹 개발에 필요한 기술

1. 프로트엔드
-디자이너와 함께 협업하여 디자인을 코드로 적용시킨다.(퍼블리싱)
-사용자에게 보이는 프론트 영역에서 서버와의 통신을 통해 데이터를 주고받는다.
-여러 라이브러리, 프레임워크를 사용한다.
(리액트, 앵귤러, 뷰, NEXT.JS, jquery,....)

2. 백엔드
- 사용자에게 보이지 않는 서버쪽에서 데이터를 주고 받는다.
- 서버 마다 사용하는 언어와 프레임워크가 다르다.
  언어 : 자바, 파이썬, JS, php ......
  프레임워크 :  스프링, 장고, 라라벨 ................

3. 기본
- 웹 개발을 할 때 기본적으로 알아야하는 언어나 기술
- HTML, CSS, JS, Git, ....

=============================================

웹(WWW)
- 인터넷에서 동작하는 서비스 중 하나이다. (인터넷 없이는 동작할 수 없다.)
- 서로 정보를 공유할 수 있는 공간이다.
- 인터넷이 기기간 연결된 큰 네트워크라면 웹은 그 네트워크 상에서 정보를 주고 받을 수 있는 공간이며 방식이다.

인터넷(Internet)
- 여러 네트워크들을 연결시켜 전세계를 연결한 네트워크 (엄청 큰 네트워크)

네트워크(통신망)
- 두 대 이상의 컴퓨터 같은 기기들이 연결되어 통신하는 체계

웹 브라우저
- 웹에 접속할 수 있는 소프트웨어

웹 페이지
- 웹 상에 존재하며 특정 정보를 담고있는 문서
- 일반 책의 페이지와는 다르게 HyperLink라는 것을 사용하여 다른 페이지로 빠르게 넘어 갈 수 있으며,
HyperLink는 HyperText에서 사용이 가능하다.
- 웹 브라우저를 통해 사용자에게 보여진다.
- 웹 페이지는 HTML, CSS, JavaScript 등으로 만들어 진다.

- 웹 페이지의 종류
* 정적 페이지 : 회사 소개 페이지처럼 누가 언제 봐도 동일한 정보를 보여주는 페이지.
* 동적 페이지 : 우리가 일상적으로 이용하는 서비스처럼 누가 보는지, 언제 보는지에 따라 다른 정보를 보여주는 페이지.

웹 사이트
- 연관된 웹 페이지들의 모음.

웹 어플리케이션 (웹 앱)
웹 사이트가 발전하면서 단순한 정적 페이지가 아닌 사용자의 요청에 따라 동적 페이지로 보여주며, 여러 서비스를 제공하게 되었다.
웹 사이트가 정적인 책의 느낌이라면 웹 앱은 마치 하나의 응용 프로그램처럼 작동을 하게되어 구분하였지만 전부 웹 사이트라고 말하기도 한다.


인터넷의 구성요소
서버 : 서비스를 제공하는 기기.
클라이언트 : 서비스를 제공받는 기기.

웹 사이트를 접속한다는 것은 웹 브라우저를 통해 서버에 요청(Request)을 전송하는 것이며 서버에서 요청을 올바르게 처리하여 다시 클라이언트에게 응답(Response)를 보내면 클라이언트는 웹 브라우저를 통해 데이터를 받아보는 것이다. 
위 과정이 해당 사이트의 서비스를 제공받는 과정이다.

인터넷의 Protocol (규약)
IP (Internet Protocol)
인터넷, 네트워크에 연결한 각 기기들을 구별할 수 있는 고유한 주소.

HTTP(HyperText Transfer Protocol)
웹에서 HTML 문서를 주고 받기위한 통신 규약.
초기에는 HTML만 주고 받았지만 지금은 JSON, XML등 많은 것들을 주고 받는다.
클라이언트와 서버가 서로 응답과 요청을 주고받는 방식이다.

웹 사이트의 구성
HTML : 웹 페이지의 뼈대를 구성한다.
CSS : HTML에 살을 입히고 꾸며준다.
JavaScript : 움직이고 생기고 없어지는 등의 동작, 반응을 수행한다.

HTML의 기본 구조

<html> -> 웹 문서의 시작과 끝을 나타냄.
	<head>
		이 페이지에 관련된 메타정보.
	</head>
	<body>
		본문 -> 실제 웹 페이지에 나타내는 내용.
	</body>
</html>
